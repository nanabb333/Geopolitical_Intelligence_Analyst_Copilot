# Analyst Workbench

The Analyst Workbench turns a free-text geopolitical scenario into a structured Markdown report for analyst review. It is designed to improve workflow productivity by packaging deterministic classification, historical analogue retrieval, observed market-reaction context, pathway framing, and executive memo language into one reusable artifact.

The workbench is not a forecasting tool, investment tool, real-time monitor, dashboard, API service, or LLM workflow.

## Workflow

```text
Question
  -> Scenario Classification
  -> Historical Analogue Retrieval
  -> Market Reaction Comparison
  -> Observed Pathway Generation
  -> Executive Memo
  -> Export Package
```

### 1. Scenario Classification

`src/classify_event.py` applies deterministic keyword rules to the submitted question. It produces a coded scenario profile with event type, policy subtype, country or region, sector, threat signal, opportunity signal, state support signal, matched terms, and caveats.

### 2. Historical Analogue Retrieval

`src/retrieve_analogs.py` compares the scenario profile against `data/events.csv`. It preserves the existing weighted scoring logic and returns the highest-scoring local historical analogues with transparent score rationales.

### 3. Market Reaction Comparison

`src/compare_market_reactions.py` summarizes observed event-study rows from `data/event_firm_returns.csv` for the retrieved analogues. These values describe historical event windows only. They are not expected returns, forecasts, probabilities, or trading signals.

### 4. Observed Pathway Generation

`src/generate_pathways.py` maps the scenario profile and retrieved analogue metadata into deterministic pathway frames. Pathways are monitoring frames for analyst discussion, not predictions.

### 5. Executive Memo

`src/generate_workbench_report.py` packages the workflow into a structured Markdown report with clearly separated sections:

- A. Scenario Classification
- B. Historical Analogues
- C. Market Reaction Comparison
- D. Observed Pathways
- E. Executive Memo

The memo is written for portfolio manager, risk committee, strategy team, or analyst-review handoff.

## Inputs

### User Query

A short geopolitical event description or analyst question.

Example:

```text
US announces a new semiconductor subsidy package.
```

### Local Data Files

- `data/events.csv`: coded historical geopolitical events and scenario metadata.
- `data/event_firm_returns.csv`: local event-study reaction rows linked by `event_id`.

The workbench reads local files only. It does not call external APIs, scrape websites, monitor live news, or use LLMs.

## Outputs

The primary output is a structured Markdown report generated by:

```bash
python3 src/run_scenario.py \
  --query "US announces a new semiconductor subsidy package." \
  --top-k 5 \
  --output results/analyst_workbench_report.md
```

The report includes:

- scenario classification table and matched deterministic terms;
- ranked historical analogues with similarity scores and retrieval rationales;
- observed market-reaction comparison for available local event-study records;
- pathway frames with historical basis, monitoring focus, and interpretation limits;
- executive memo language suitable for analyst handoff;
- export-package metadata describing the reusable artifact and source inputs.

## Intended Use

The Analyst Workbench is intended for:

- accelerating first-pass scenario triage;
- making historical analogue retrieval repeatable;
- preparing committee-ready Markdown notes;
- giving analysts a structured starting point for review;
- documenting why analogues were selected and where the evidence is limited.

It should be used as a deterministic workflow automation layer. Analysts remain responsible for validating event fit, reviewing source context, checking data quality flags, and deciding whether the historical comparison is appropriate for the decision context.

## Safety Boundaries

The workbench does not:

- forecast future market levels or policy outcomes;
- estimate probabilities;
- provide investment recommendations;
- produce trading signals;
- generate expected returns;
- ingest live news;
- call APIs;
- call LLMs;
- create dashboards or real-time monitors.

Similarity scores are retrieval scores only. Market-reaction summaries are historical observations only.
